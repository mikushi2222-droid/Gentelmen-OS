# 00. Обзор проекта

## 1. Что мы строим

**Gentleman OS** — личное офлайн-приложение для одного пользователя. Это не
«цитатник из книг про стиль», а **личная система принятия решений** для мужчины:
что надеть, что купить, что улучшить, как выглядеть собраннее, как не ошибаться
с посадкой, цветами и дресс-кодом.

Главная ценность — **скорость ежедневного применения**. Для личного инструмента
выигрывает не ширина функционала, а быстрый ответ на бытовой вопрос «что делать
сегодня».

## 2. Почему именно так

Приложение строится как **offline-first, local-only, feature-first** продукт.
Для одного пользователя это даёт три преимущества:

1. **Приватность** — данные не покидают устройство.
2. **Быстродействие** — UI не ждёт сеть.
3. **Отсутствие лишней сложности** — нет сервера, аутентификации, синхронизации.

Flutter официально рекомендует offline-first подход для приложений, которым
важно работать без сети, и описывает разбиение на слои UI / logic / data с
чёткими правилами коммуникации между ними. Это хорошо ложится на личный инструмент.

## 3. Целевой пользователь

Один человек — владелец приложения. Конкретные параметры (рост, вес, талия,
объёмы) задаются в профиле и влияют на эвристики рекомендаций. Приложение
оптимизировано под крупную фигуру по умолчанию, но правила параметризованы и
подстраиваются под данные профиля.

## 4. Ключевое решение проекта

> Самое сильное решение здесь — **не ИИ, а локальное ядро с хорошими правилами.**
> Если оно качественное, приложение полезно даже без сети и без интеграций.

Фундамент — это цикл:

```
ввод данных → рекомендация → действие → сохранение результата
```

ИИ-слой подключается **поверх** этого ядра и опционален: реализован как порт
`AiAdvisor` с оффлайн-движком по умолчанию и облачной реализацией через RouterAI
(включается вводом ключа). Подробнее — [14-ai-integration.md](14-ai-integration.md).

## 4a. Реализованные модули (актуально, июнь 2026, схема v6)

### Ядро и инфраструктура
- **Scaffold** — Material 3, тёмная тема (seed `#C9A84C`), `ShellRoute` с 5 табами
- **Drift БД v6** — 14 таблиц, 10 транзакционных DAO, миграции v1→v6, сидовый контент
- **Экспорт/очистка** — JSON-бэкап через `share_plus`, полная очистка данных
- **Журнал отладки**, **защищённое хранение ключа ИИ** (`flutter_secure_storage`)

### Dashboard
- **Gentleman Score** (5-компонентная формула: стиль 25%, фитнес 25%, привычки 30%, чтение 10%, здоровье 10%)
- **Ежедневные миссии** (3 задачи дня: замер, гардероб, привычка; проверка через реальный DB)
- **Аниме-маскот** (`MascotAvatar`) — меняет настроение (`sleeping/neutral/pleased/proud`) от Score
- **Мини-плитка Индекса здоровья** — переход в раздел здоровья
- **"Надеть сегодня" urgency strip** — горизонтальная лента вещей с `WearUrgency.today/soon`
- **Быстрый доступ** — 9 кнопок к ключевым разделам

### Гардероб
- CRUD вещей с фото (camera/gallery), cost-per-wear, оценка удобства
- **Прогноз носки `WearForecast`** — 5 статусов (`today/soon/onRotation/offSeason/retired`)
- **Карточка прогноза** на экране детали вещи
- Фильтр по категории, urgency-сортировка (по срочности / по дате добавления)
- **Поиск** по названию и бренду (client-side, inline в AppBar)
- Статистика гардероба (стоимость, самая носимая, лучшая цена/носка)
- ИИ-анализ фото вещи (vision через RouterAI)

### Outfit Builder
- Генератор образов по поводу/погоде/дресс-коду/сезону (scoring: посадка × 0.30, повод × 0.25, погода × 0.20, цвет × 0.15, комфорт × 0.10)
- Сохранение образа с полным JSON score breakdown
- Список образов со score-индикатором
- **Детальный экран** — разбивка score по 5 осям + объяснение тезисно
- **"Надеть весь образ"** — batch-инкремент носки для всех вещей образа

### База знаний
- Статьи в Markdown (поиск по тексту/тегам, статус «Прочитано»)
- Закладки и избранное; фильтр-чипы «Закладки» и «Избранное» в категориях
- **Время чтения** (~N мин) — чип в шапке каждой статьи
- ИИ-рекомендации «Рекомендуем прочитать» на основе гардероба

### Фитнес
- Логи замеров (вес, талия, грудь, плечи)
- Графики fl_chart по метрикам
- **Тренд-дельты** (стрелки ↑↓ и разница к предыдущему замеру)

### Привычки
- CRUD привычек, отметка выполнения, стрик
- **7-дневный calendar strip** на каждой привычке (зелёные/пустые кружки)

### Мужское здоровье
- 16 маркеров (тестостерон, ПСА, витамин D, ферритин, холестерин и др.)
- Цветовой статус (норма/внимание/риск), референсные диапазоны
- Индекс здоровья [0–100], графики динамики
- **ИИ-разбор маркеров** с доказательными рекомендациями (RouterAI + веб-поиск)
- Дисклеймер «не является медицинской рекомендацией»

### RPG и геймификация
- XP по 8 типам (`XpType`: `style`, `fitness`, `etiquette`, `reading`, `career`, `finance`, `general`, `health`)
- Gentleman Score агрегирует 5 компонентов (стиль, фитнес, привычки, чтение, здоровье)
- Уровни и прогресс-бар, навыки с историей XP
- Достижения (разблокированные/закрытые)
- Ежедневные миссии со связью в выполнение реальных действий

### Покупки
- Список желаний с правилом 48 часов (cooldown для импульсивных покупок)
- **5 табов по статусу** (Все / Хочу / Планирую / Куплено / Отклонено)
- Приоритеты, бюджет, сводка планируемых расходов

### ИИ-слой
- Абстракция `AiAdvisor` — `LocalAiAdvisor` (оффлайн) или `RouterAiClient` (облако)
- Активация через ввод ключа RouterAI в настройках
- Советник по гардеробу (`styleAdviceProvider`), ИИ-разбор здоровья, анализ фото

## 5. Что НЕ делаем (явные анти-цели)

- ❌ Сервер / backend
- ❌ Аутентификация и аккаунты
- ❌ Мультипользователь
- ❌ Социальная лента, чат для всех
- ❌ Маркетплейс
- ❌ Облачная синхронизация (в v1)
- ❌ Сложный чат-бот на первом этапе

Эти вещи только раздувают проект и мешают ядру.

## 6. Нефункциональные требования

### Производительность
- Быстрый холодный старт (цель < 2 с до интерактивного Dashboard).
- Основной UI никогда не блокируется сетью.
- Изображения гардероба кешируются и масштабируются (thumbnails).
- Длинные списки рендерятся лениво (`ListView.builder` / `SliverList`).

### Надёжность
- Приложение не теряет данные при перезапуске.
- Локальная БД имеет версионирование и миграции.
- Crash-safe записи (транзакции Drift).
- Экспорт/импорт всех данных обязателен.

### Приватность
- Все данные — локально.
- Внешние ключи — только если пользователь сам их добавит.
- Ничего не отправляется наружу без явного действия.

### Доступность
- Крупные заголовки.
- Контрастные цвета.
- Удобные touch targets (≥ 48dp).
- Тёмная тема по умолчанию.

### Тестируемость
- Unit-тесты для логики (use cases, scoring, репозитории).
- Widget-тесты для ключевых экранов.
- Golden-тесты для Dashboard / Wardrobe / Outfit Builder (по возможности).
- Тесты экспорта/импорта и миграций БД.

## 7. Метрики успеха (для себя)

- Время от запуска до «получить образ на сегодня» < 15 секунд.
- 100% данных переживают переустановку через экспорт/импорт.
- Любая рекомендация показывает «почему» (объяснимость).
- Ежедневное использование без раздражения от сложности.

## 8. Платформа

- **Android** (основная цель v1).
- Flutter позволяет позже расширить на iOS/desktop без переписывания ядра, но в
  v1 это не приоритет.
