# 01. PRD — Product Requirements Document

## 1. Vision

Личный цифровой помощник «джентльмена»: помогает со стилем, одеждой, грумингом,
этикетом, трекингом формы, чтением, привычками и личным прогрессом. Полностью
офлайн, приватно, для одного пользователя.

## 2. Цели и не-цели

### Цели
- Дать быстрый ответ «что надеть сегодня» из реального гардероба.
- Хранить и анализировать гардероб (включая cost-per-wear).
- Вести замеры тела и показывать прогресс.
- Накапливать личную базу знаний по стилю/этикету.
- Мотивировать через RPG-механику.
- Помогать с осознанными покупками.

### Не-цели (v1)
- Сеть, аккаунты, мультипользователь, соцфункции, маркетплейс, облако, чат-бот.

## 3. Персона

Единственный пользователь — владелец. Параметры тела хранятся в профиле и
напрямую влияют на эвристики (например, крупная фигура → структурированные формы,
средняя/высокая посадка, спокойные цвета, плотные ткани).

## 4. Функциональные требования (фичи)

### 4.1 Dashboard (экран дня)
**Цель:** за один взгляд понять состояние дня и получить действие.

Показывает:
- приветствие;
- **Gentleman Score** (агрегированный показатель «собранности»);
- 3 задачи дня (missions);
- рекомендацию/образ дня;
- быстрый доступ к гардеробу, профилю, образам, замерам, заметкам.

Acceptance:
- Экран рендерится из локальных данных без задержек.
- Если данных мало — показываются onboarding-подсказки.

### 4.2 Personal Profile
**Поля:** рост, вес, талия, грудь, бёдра, плечи, шея, размер обуви,
предпочтения по стилю, любимые цвета, бюджетные предпочтения, запреты/неудобства.

Функции:
- редактирование замеров;
- предпочтения и цели;
- история изменений (через MeasurementLog);
- производные рекомендации (derived recommendations) на основе параметров.

Acceptance:
- Изменение замера создаёт запись истории.
- Производные рекомендации пересчитываются при изменении профиля.

### 4.3 Digital Wardrobe (гардероб)
**Карточка вещи (ClothingItem):** имя, категория, бренд, размер, цвет, материал,
сезон, посадка (fit), цена, дата покупки, фото (imagePath), заметка, состояние
(condition), оценка удобства (rating).

Категории: shirts, polos, t-shirts, trousers, jeans, blazers, jackets, coats,
shoes, accessories.

Функции:
- добавление вещи с фото;
- поиск, фильтр (категория/сезон/цвет/бренд), сортировка;
- **cost-per-wear** (цена / число надеваний);
- карточка детали вещи.

Acceptance:
- Фото сохраняется локально, отображается thumbnail в сетке.
- Cost-per-wear обновляется при отметке носки (через Outfit/wear-лог).

### 4.4 Outfit Builder (генератор образов)
**Входы:** повод (occasion), погода, температура, сезон, дресс-код, настроение,
ограничения.

**Выход:** 1–3 комплекта из реального гардероба с объяснением выбора.

Функции:
- задать параметры;
- получить ранжированные подборки;
- сохранить комплект (Outfit);
- отметить «что сработало» (rate после носки).

Acceptance:
- Каждый предложенный образ имеет итоговый score и разбивку по осям.
- Образ собирается только из доступных (не «в стирке») вещей.

### 4.5 Knowledge Base (энциклопедия)
Локальные статьи в Markdown.

Категории: style, etiquette, grooming, fabrics, shoes, suits, casual, health,
discipline, reading.

Функции: поиск, теги, избранное, закладки.

Примеры статей:
- как должен сидеть пиджак;
- какие воротники подходят;
- какие ткани лучше;
- что не покупать;
- как сочетать цвета;
- как выглядеть дороже без дорогих вещей.

Acceptance:
- Контент рендерится из Markdown.
- Поиск ищет по заголовку, тегам и тексту.

### 4.6 Fitness & Measurements
**Логи:** вес, талия, грудь (и др. замеры), шаги, тренировки, заметки о самочувствии.

Функции: ввод значений, графики прогресса, история.

Acceptance:
- Каждая запись датирована.
- Графики строятся по выбранной метрике и периоду.

### 4.7 Gentleman RPG
**Механика:** XP, уровни, навыки (style, fitness, etiquette, reading, career,
finance), достижения, стрики.

Назначение — связать повседневные действия с ощущением прогресса (мягкая мотивация).

Acceptance:
- Действия в приложении начисляют XP через XpEvent.
- Уровень и навыки вычисляются детерминированно из событий.

### 4.8 Purchase Advisor (советник по покупкам)
**PurchaseWish:** имя вещи, категория, приоритет, бюджет, причина, статус.

Функции: хранить желаемое, сравнивать приоритеты и бюджет, видеть «что купить дальше».

Acceptance:
- Список сортируется по приоритету/бюджету.
- Статус (wish / planned / bought / rejected) меняется и сохраняется.

## 5. Бизнес-правила (эвристики)

- Избегать skinny fit для крупной фигуры.
- Предпочитать среднюю/высокую посадку для большой талии.
- Рекомендовать структурированные лацканы умеренной ширины.
- Предпочитать плотные/тяжёлые ткани тонким блестящим.
- Приоритет: пропорциональность, комфорт, повторяемость.
- Scoring образа использует: fit, color harmony, occasion match, weather match,
  comfort.
- Рекомендации должны быть **объяснимыми**.

## 9. Out of scope для v1 (отложено)

Синхронизация с сервером, командная работа, публикация контента, маркетплейс,
соцлента, сложный чат-бот. Всё это — после того, как личный цикл работает идеально.
