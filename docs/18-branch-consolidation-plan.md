## Архивный план (2026-06-23) — для справки

Ниже сохранён исходный план для понимания контекста и принятых решений.

### Почему нельзя было просто слить

Ветки имели **две независимые истории** с пересекающимися фичами:
- `main`: schema v5, чистая база (analyze 0/0/0), закалённый CI;
- `gentleman-os-plan-cgm6hh`: schema v7, своя линия миграций.

Прямой `git merge` дал бы конфликты в схеме БД (v5 ↔ v7), pubspec и доках
практически в каждом файле.

### Принятое решение

Не мёржить историю, а **реализовать `git rebase`** — воспроизвести 23 коммита
`gentleman-os-plan-cgm6hh` поверх свежего `main`, с ручным разрешением
конфликтов в ключевых файлах. Результат — одна чистая линия без дублирующихся
историй.

### Что было разрешено при ребейзе

| Файл | Конфликт | Решение |
|------|----------|---------|
| `wear_forecast.dart` | Две несовместимые реализации WearForecast | Объединены: единый класс + обе функции |
| `app_database.dart` | Потеря schema v7 при взятии HEAD | Восстановлена отдельным коммитом |
| `dashboard_providers.dart` | Несколько раундов: healthXp, articlesRead, дубли | Вычищены все дублирующиеся объявления |
| Документация | Множественные конфликты | HEAD (main) как основа, V3.x-контент из ветки |
