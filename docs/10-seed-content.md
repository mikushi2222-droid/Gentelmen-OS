# 10. Сидовый контент и стартовые данные

При первом запуске (`onCreate` миграции) БД засевается стартовым контентом, чтобы
приложение было полезным «из коробки» и экраны не были пустыми.

## 1. Что сеется

| Данные | Объём | Источник |
|--------|-------|----------|
| Knowledge-статьи | 8–12 | `assets/knowledge/*.md` |
| Достижения (закрытые) | весь каталог | код (`achievement_rules`) |
| Привычки-шаблоны | 3–5 (неактивные) | `assets/seed/habits.json` |
| Пример вещей гардероба | 0 (пусто, опц. демо) | `assets/seed/demo_items.json` |
| Daily missions | генерируются | правила |

> Профиль НЕ заполняется демо-данными — пользователь вводит реальные параметры.

## 2. Стартовые статьи базы знаний (Markdown)

Минимальный набор, покрывающий ежедневные вопросы (категории из data-model):

1. **Как должен сидеть пиджак** (style/suits) — плечи, длина рукава, прилегание.
2. **Какие воротники подходят** (style) — под форму лица/шеи.
3. **Какие ткани лучше** (fabrics) — плотные/натуральные vs тонкие/блестящие.
4. **Что не покупать** (style) — частые ошибки, импульсивные покупки.
5. **Как сочетать цвета** (style) — нейтральные базы, акценты, контраст.
6. **Как выглядеть дороже без дорогих вещей** (style) — посадка, уход, базовая палитра.
7. **Посадка брюк для крупной фигуры** (style) — средняя/высокая посадка, прямой крой.
8. **Базовый груминг** (grooming) — минимальный ежедневный набор.
9. **Этикет: базовые правила** (etiquette).
10. **Дисциплина и привычки** (discipline) — связь с RPG-слоем.

Каждая статья: заголовок, категория, теги, Markdown-тело, опц. источник.

## 3. Пример seed-статьи (формат файла)

`assets/knowledge/fit_blazer.md`:
```markdown
---
id: fit-blazer
title: Как должен сидеть пиджак
category: suits
tags: [посадка, пиджак, плечи]
sourceRef: null
---

## Плечи
Шов пиджака заканчивается там, где заканчивается ваше плечо...

## Длина рукава
Из-под рукава видно 1–1.5 см манжеты рубашки...

## Прилегание
Пиджак должен застёгиваться без заломов «X» на пуговице...
```

Фронтматтер парсится при сиде → `KnowledgeArticle`.

## 4. Каталог достижений (seed)

См. [06-rpg-and-gamification.md](06-rpg-and-gamification.md) §6. Все ачивки
создаются как `unlocked = false` и разблокируются правилами по мере действий.

## 5. Шаблоны привычек (`assets/seed/habits.json`)

```json
[
  { "title": "10 000 шагов", "target": 10000, "period": "daily", "active": false },
  { "title": "Чтение 20 минут", "target": 1, "period": "daily", "active": false },
  { "title": "Тренировка", "target": 3, "period": "weekly", "active": false },
  { "title": "Уход за обувью", "target": 1, "period": "weekly", "active": false }
]
```

Создаются неактивными — пользователь сам активирует нужные.

## 6. Демо-вещи (опционально, `assets/seed/demo_items.json`)

Для демонстрации Outfit Builder можно (опционально, по флагу) посеять 6–8 базовых
вещей: navy chinos, серые брюки, белая рубашка, голубая рубашка, тёмное поло,
тёмно-синий блейзер, коричневые туфли, белые кеды. Это позволяет сразу показать
работу движка рекомендаций. По умолчанию выключено, чтобы не засорять реальный
гардероб.

## 7. Идемпотентность сида

- Сид выполняется только в `onCreate` (один раз).
- Повторный сид при апгрейде — только для нового обязательного контента, с
  проверкой по уникальным кодам (`achievement.code`, `article.id`), без дублей.

## 8. Локализация контента

- Стартовые статьи — на русском (язык пользователя).
- Технические enum/коды — на английском (стабильные идентификаторы).
