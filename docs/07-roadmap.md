# 07. Дорожная карта и спринт-бэклог

## 1. Порядок реализации (milestones)

Принцип: **приложение остаётся полностью запускаемым на каждом этапе.** После
каждого milestone — короткое резюме сделанного, без поломки существующего.

| # | Milestone | Результат |
|---|-----------|-----------|
| 1 | Project scaffold | пустой запускаемый Flutter-проект, CI-команды, build_runner |
| 2 | Theme & design system | Material 3, тёмная тема, общие виджеты |
| 3 | Local DB & entities | Drift-таблицы, миграции v1, репозитории, сид |
| 4 | Profile | профиль + история замеров + производные рекомендации |
| 5 | Wardrobe | CRUD вещей, фото, фильтры, cost-per-wear |
| 6 | Outfit Builder | движок scoring + генерация + сохранение/оценка |
| 7 | Knowledge Base | статьи Markdown, поиск, избранное, закладки |
| 8 | Fitness tracking | логи замеров, графики |
| 9 | RPG system | XP, уровни, навыки, ачивки, Gentleman Score, missions |
| 10 | Export/Import | JSON-бэкап, импорт, очистка |
| 11 | Tests | unit/widget/golden/migration покрытие |
| 12 | Polish & refactor | доступность, производительность, рефактор |
| 13 | Premium redesign | палитра шампань-золото, блочная Главная, luxury-anime декор |
| 14 | Navigation v2 | три уровня в нижнем меню (Главная/Гардероб/Здоровье/Биохакинг/Профиль) |
| 15 | Health expansion | гормоны, метаболизм, сердце, сон, анализы, биологический возраст |
| 16 | Biohacking | оптимизация, узкие места, протоколы, добавки (доказательность A/B/C) |
| 17 | Wardrobe wear forecast | прогноз износа вещи (дифференциатор) |

## 2. Привязка к этапам (как в исходном плане)

- **Этап 1:** проект, тема, навигация, локальная БД, профиль, гардероб → milestones 1–5.
- **Этап 2:** outfit builder, истории замеров, базовая энциклопедия → milestones 6–8.
- **Этап 3:** habits, RPG, рейтинг образов → milestone 9.
- **Этап 4:** экспорт/импорт, аналитика, красивые графики, будущий AI-слой → milestones 10–12 + v2.

## 3. Sprint 1 backlog (детально)

Цель спринта 1: **запускаемое приложение с навигацией, темой, БД, профилем и
каркасом гардероба.**

профиль редактируется и сохраняется в БД, можно добавить и увидеть вещь, тесты БД
зелёные.

## 7. Будущее (v2+, явно отложено)

- Опциональный локальный/онлайн AI-слой (объяснимые подсказки, разбор фото вещи).
- Облачный бэкап (по желанию пользователя, с шифрованием).
- iOS-сборка.

> Правило: AI и облако подключаются **только после** того, как локальный цикл
> «ввод → рекомендация → действие → результат» работает идеально.

## 8. Риски и митига

| Риск | Митигация |
|------|-----------|
| Раздувание scope | жёсткие анти-цели (см. PRD §2) |
| Сложность кодгена | один источник правды по командам в README |
| Перебор комбинаций в генераторе | greedy/beam, ограничение кандидатов |
| Потеря данных | миграции + экспорт/импорт + транзакции |
| Снижение мотивации | RPG-слой, но без превращения в «игру ради игры» |
