# 06. RPG-слой и геймификация

Система уровней и XP нужна **не ради игры**, а чтобы связывать повседневные
действия с ощущением прогресса. Для личного приложения это работает как мягкая
мотивация.

## 1. Принцип

- Единственный источник истины — поток `XpEvent`.
- Уровень, навыки и стрики **вычисляются** из событий (детерминированно), а не
  хранятся как изменяемые поля. Это упрощает тесты и исключает рассинхрон.

## 2. Навыки (skills)

Шесть направлений, каждое набирает XP независимо:

| Навык | Источник XP (примеры действий) |
|-------|-------------------------------|
| `style` | собрать образ, оценить образ, добавить вещь осознанно |
| `fitness` | записать замер, отметить тренировку, шаги |
| `etiquette` | прочитать статью по этикету |
| `reading` | отметить чтение/прочитанную статью |
| `career` | выполнить связанную задачу/привычку |
| `finance` | закрыть PurchaseWish осознанно, уложиться в бюджет |

Плюс `general` — для действий, не привязанных к навыку, и `health` — XP за
внесение маркеров мужского здоровья. Полный enum (`XpType`, 8 значений):
`style, fitness, etiquette, reading, career, finance, general, health`.

## 3. Начисление XP

Каждое значимое действие создаёт `XpEvent { type, amount, reason, createdAt }`.

Примерная таблица наград (значения заданы в `core/services/xp_service.dart`):

| Действие | type | amount |
|----------|------|--------|
| Добавить вещь в гардероб | style | 10 |
| Собрать образ | style | 15 |
| Оценить образ после носки | style | 20 |
| Записать замер | fitness | 15 |
| Отметить тренировку | fitness | 25 |
| Прочитать статью | reading/etiquette | 10 |
| Выполнить привычку (день) | general/career | 10 |
| Удержать стрик 7 дней | general | 50 (бонус) |
| Закрыть покупку в бюджете | finance | 20 |

## 4. Формула уровня

XP суммируется глобально (общий уровень) и по каждому навыку (уровень навыка).

Кривая уровней (растущая стоимость):
```
xpForLevel(n) = base * n^1.5        // base = 100
totalXpToReach(L) = Σ_{n=1..L-1} xpForLevel(n)
```

Текущий уровень = максимальный L, при котором `totalXpToReach(L) <= totalXp`.
Прогресс-бар = (totalXp − totalXpToReach(L)) / xpForLevel(L).

Чистая функция:
```dart
LevelInfo computeLevel(int totalXp) { ... }   // юнит-тестируется
```

## 5. Стрики (streaks)

- Стрик привычки — число подряд идущих периодов с выполнением (из `habit_logs`).
- Глобальный «daily streak» — число дней подряд с любой активностью.
- Стрик пересчитывается из логов (кеш в `Habit.streak` для скорости).
- Разрыв стрика — пропущенный период обнуляет счётчик (опц. «заморозка» на 1 день).

## 6. Достижения (achievements)

Декларативный список правил-условий; при каждом изменении состояния проверяются.

Примеры:
| code | Условие |
|------|---------|
| `first_outfit` | собран первый образ |
| `wardrobe_25` | 25 вещей в гардеробе |
| `streak_7` | стрик 7 дней |
| `streak_30` | стрик 30 дней |
| `down_2cm_waist` | талия снизилась на 2 см от базовой |
| `bookworm_10` | прочитано 10 статей |
| `budget_master` | 5 покупок закрыты в бюджете |
| `level_10` | достигнут 10 уровень |

Реализация: список `AchievementRule { code, check(state) -> bool }`, идемпотентная
разблокировка (ставит `unlocked=true`, `unlockedAt`).

## 7. Gentleman Score (для Dashboard)

Агрегированный показатель «собранности» за последние 7 дней. Чистая функция
`computeGentlemanScore` (в `features/rpg/domain/level_calculator.dart`) — это
**взвешенная сумма 5 компонентов**, каждый нормирован в [0, 1], итог × 100:

```
gentlemanScore = clamp(0..100,
    style    * 0.25     // styleXpLast7d   / 50  → max
  + fitness  * 0.25     // fitnessXpLast7d / 50  → max
  + habits   * 0.30     // habitsCompleted / habitsTotal (0 при total == 0)
  + reading  * 0.10     // articlesReadLast7d / 3 → max
  + health   * 0.10     // healthXpLast7d  / 30  → max
)
```

- Веса: привычки 30%, стиль 25%, фитнес 25%, чтение 10%, здоровье 10%.
- Детерминирован при фиксированном состоянии (покрыт `test/unit/rpg/`).
- Маскот (`MascotAvatar`) меняет настроение от Score: `sleeping < 20`,
  `neutral < 50`, `pleased < 80`, `proud ≥ 80`.

## 8. Задачи дня (Daily Missions)

- 3 задачи в день на Dashboard (`DailyMission`).
- Генерируются по простым правилам: «нет замера за неделю → задача замерить»,
  «нет образа на сегодня → собрать образ», «непрочитанная статья → прочитать».
- Выполнение задачи → начисляет XP и закрывает миссию.

## 9. Слой в архитектуре

Фактическое размещение (июнь 2026):

```
features/rpg/
├── domain/
│   └── level_calculator.dart      // computeLevel + computeGentlemanScore
├── application/
│   └── rpg_providers.dart         // провайдеры уровня/Score/навыков
└── presentation/
    └── rpg_screen.dart

core/services/
├── xp_service.dart                // начисление XpEvent (таблица наград)
└── achievement_service.dart       // декларативные правила ачивок, разблокировка

features/dashboard/domain/
└── mission_generator.dart         // generateDailyMissions (см. §8)
```

> Начисление XP и проверка ачивок вынесены в `core/services/` (кросс-фичевые),
> а `gentleman_score`/уровни — чистые функции в `rpg/domain/level_calculator.dart`.
> DAO для `xp_events`/`achievements` — `core/db/daos/rpg_dao.dart`.

## 10. Тестирование

- `computeLevel` — таблица вход/выход (юнит).
- Правила ачивок — каждое условие отдельно.
- Стрики — последовательности дат (включая разрывы).
- Gentleman Score — детерминирован при фиксированном состоянии.
